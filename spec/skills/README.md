### For AI Agents Contributing to A'Space OS

When you join this project (via GitHub, Gemini CLI, Claude, etc.), load the relevant skills to understand how to interact with the infrastructure.

**Example workflow:**

1. **Read the project context:**
   - Load `conductor.md` to understand Conductor workflow
   - Review `conductor/product.md` for project goals

2. **Set up your environment:**
   - Ensure you have access to necessary credentials (`.env` files)
   - Verify MCP connections are configured

3. **Start a new task:**
   - Use Conductor to create a track: `/conductor:newTrack "Your feature"`
   - Load relevant MCP skills as needed (e.g., `supabase.md` for database work)

4. **Implement the feature:**
   - Follow the `plan.md` generated by Conductor
   - Use MCP skills for specific tasks (deploy, query, automate)

5. **Sync your work:**
   - Load `mcp/github.md` to commit and push changes
   - Update `conductor/tracks.md` with status

## ðŸš€ Quick Start for New Contributors

### 1. Clone the Repository
```bash
git clone https://github.com/amadeus/aspaceos-a0-amadeus-cockpit.git
cd aspaceos-a0-amadeus-cockpit
```

### 2. Load Conductor Skill
```bash
# Read the skill
cat spec/skills/conductor.md

# Install Conductor extension
gemini extensions install https://github.com/gemini-cli-extensions/conductor --auto-update
```

### 3. Initialize Your Session
```bash
# Setup Conductor (if first time)
/conductor:setup

# Or review existing context
cat conductor/product.md
cat conductor/tech-stack.md
```

### 4. Start Contributing
```bash
# Create a track for your feature
/conductor:newTrack "Your feature description"

# Implement following the plan
/conductor:implement

# Commit your work (load github.md skill)
git add .
git commit -m "feat: your feature"
git push origin develop
```
