# SCOPE : A'SPACE OS INTERFACE

## In Scope (Ce que l'interface FERA)

### Phase 1 (MVP - Minimum Viable Product)
- [ ] Affichage des 8 Life Domains (scores visuels)
- [ ] Terminal A0 Command (commandes root)
- [ ] Liste Projects PARA (statut des 12WY actifs)
- [ ] Business Pulse (TMI, TVR, dernier rapport Jerry)

### Phase 2 (Enhancement)
- [ ] Sunday Uplink automatique (génération + envoi)
- [ ] Visualiseur de contrats (Intent/Decision/Order/Pulse)
- [ ] Audit Rick en temps réel (warnings biomimétiques)

### Phase 3 (Advanced)
- [ ] Mode "Fractal Zoom" (de la forêt PARA à la feuille)
- [ ] Integration n8n workflows (trigger depuis l'UI)
- [ ] Voice commands (via Gemini Live)

## Out of Scope (Ce que l'interface ne fera JAMAIS)

### Anti-Features (Interdits par Design)
❌ Social media integration  
❌ Gamification (badges, streaks, points)  
❌ Push notifications (sauf Red/Orange signals)  
❌ Analytics tracking invasif  
❌ Addictive patterns (infinite scroll, auto-play)  

### Non-Priorités (Peut-être Plus Tard, Peut-être Jamais)
* Mobile app native (le web suffit)
* Offline mode (connexion assumée)
* Multi-user (mono-user by design)

---

> **Règle d'Or :** "Si une feature peut distraire l'Architecte, elle n'a pas sa place."
